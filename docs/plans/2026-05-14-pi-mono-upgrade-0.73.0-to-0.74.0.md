# pi-mono SDK Upgrade 0.73.0 → 0.74.0 (Package Scope Migration)

## Overview

Upgrade `@mariozechner/pi-coding-agent`, `@mariozechner/pi-ai`, and `@mariozechner/pi-agent-core` from `^0.73.0` to `^0.74.0`. The single substantive change in 0.74.0 is a package scope rename: `@mariozechner/*` → `@earendil-works/*`. The 0.74.0 release also rolls up the 0.73.1 bugfix release, none of which require code changes from Akmatori (HTML export XML stripping is automatic; OAuth-select / JSONC models.json / TUI fixes are not used by the agent-worker).

Verified against `/opt/pi-mono` at tag `v0.74.0`:
- `packages/coding-agent/package.json`: `"name": "@earendil-works/pi-coding-agent"`, version `0.74.0`
- `packages/ai/package.json`: `"name": "@earendil-works/pi-ai"`, version `0.74.0`
- `packages/agent/package.json`: `"name": "@earendil-works/pi-agent-core"`, version `0.74.0`
- Public API exports are unchanged except for two additive OAuth select types (`OAuthSelectOption`, `OAuthSelectPrompt`) not used by Akmatori
- `models.generated.ts` removed a few unused third-party model entries (k2p6, allenai/olmo-3.1, meta-llama/llama-3, mimo-v2-*) — none referenced in Akmatori's `MODEL_SUGGESTIONS`

Therefore this upgrade is purely mechanical: rename imports and dependency keys.
